# Employee Accommodation Management System (EAM)

## Overview
Web-based system for managing company employee accommodation using QR-based room identification. Built for HR & Administration departments.

## Architecture
- **Frontend:** React + TypeScript + Tailwind CSS + shadcn/ui
- **Backend:** Node.js + Express
- **Database:** PostgreSQL with Drizzle ORM
- **Auth:** Session-based with bcrypt password hashing
- **QR:** Server-side QR code generation with `qrcode` package

## User Roles
- **Super Admin:** Full access including user management
- **HR User:** Can manage employees and rooms, cannot manage system users

## Key Features
- Dashboard with occupancy stats and recent transfers
- Employee CRUD with room assignment and transfer tracking
- Room CRUD with auto-generated QR codes
- Public QR page (no login required) showing room details and employee cards
- User management (admin only)
- Session-based authentication with role-based access control
- Employee profile photo upload (JPG/PNG, 2MB limit)
- Advanced Export & Reporting system (see Export section below)
- Export activity logging in export_logs table

## Default Login

## Database Schema
- `users` - System users (id, name, email, password, role, status)
- `rooms` - Accommodation rooms (id, room_number, building, floor, capacity, status, qr_hash)
- `employees` - Employee records (id, employee_id_no, name, iqama, mobile, department, company, profile_image, status, room_id)
- `transfer_logs` - Room transfer history (id, employee_id, from_room_id, to_room_id, transferred_at, transferred_by)
- `export_logs` - Export activity tracking (id, user_id, export_type, format, created_at)

## Export & Reporting System

### Master Summary Header (all exports)
Every export document includes at the top:
- Report Title
- Generated By (logged-in user name)
- Date & Time Generated
- Total Employees, Total Rooms, Occupied Rooms, Available Rooms
- When filters are applied, summary reflects filtered totals only

### Export Endpoints
- `GET /api/export/employees/excel` - Excel with summary header + employee table + "Room Building" column
- `GET /api/export/employees/pdf` - Landscape PDF with summary box + styled table + page numbers + system footer
- `GET /api/export/rooms/excel` - Excel with room-by-room sections, each listing employees inside that room
- `GET /api/export/rooms/pdf` - PDF with room cards showing room details + employee bullet list per room
- `GET /api/export/rooms/qr-pdf?layout=single|grid` - QR code PDF with summary header (1 or 4 per page)

### Filter Support (all export endpoints)
Query params: `?department=X&company=X&building=X&roomId=X&status=X`
- Filters apply to both data and summary totals
- Building filter constrains rooms and their employees
- Department/company filters constrain employees
- Empty rooms show "No employees assigned to this room."
